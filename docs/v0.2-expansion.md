# v0.2 instance expansion

The v0.2 research task keeps the 25 reviewed family contracts and evaluates each under four
deterministic scenario seeds. This yields 100 business-decision instances. Each instance has one
clean and one named-perturbation sample, yielding 200 evaluation samples.

`data/task_specs/v0.2-instances.json` is the machine-readable catalog. It records the stable instance
ID, family ID, seed, category, difficulty, horizon, prompt, clean sample ID, perturbed sample ID, and
perturbation. It is regenerated with:

```bash
decision-agent-bench export-instances data/task_specs/v0.2-instances.json
```

## Why expand by scenario seed

Repeating one fixed database risks measuring memorization of entity values rather than stable
decision behavior. The four seeds preserve designed structural facts—such as the R03 decline,
P003 recall, C00001 refund cluster, and top-three P001 allocation stores—while varying non-critical
transaction counts and forecasts. Tests verify those answer contracts for every published seed.

The expansion increases statistical replication without pretending that near-identical scenario
instances are 100 independent task concepts. Results must cluster or pair by family and seed where
appropriate. The original v0.1 25-family/50-sample task remains registered so historical results do
not silently change.

`configs/experiments/v0.2.template.json` defines the complete eight-baseline, two-variant,
three-repetition architecture and ablation grid. Run `estimate-experiment` before enabling provider
models: one enabled model requires 4,800 sample executions, and three require 14,400.
