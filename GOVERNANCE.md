# Project governance

DecisionAgentBench uses maintainer review with evidence-based change control. Maintainers approve
releases, task versions, scorer and simulator changes, public results, security responses, and new
maintainers. Routine code and documentation changes require one approving maintainer; changes that
can affect benchmark results require an issue, tests, protocol/changelog review, and two independent
approvals when two unconflicted maintainers are available.

Published artifacts are append-only. Corrections identify affected versions and preserve original
results. Leadership decisions, funding or provider conflicts, and benchmark-policy changes are
recorded publicly unless security or privacy requires a private record.

New maintainers are selected for sustained, high-quality contributions and responsible review.
They may step down at any time and may be removed for inactivity, undisclosed material conflict,
security negligence, or code-of-conduct violations through an unconflicted maintainer decision.

The initial repository owner acts as tie-breaker until at least three maintainers exist. This is a
temporary bootstrap rule, not a claim of authority over independent result interpretations or
forks permitted by the MIT License.
